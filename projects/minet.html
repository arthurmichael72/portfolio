Product Design & UX Architecture

Overview

MiNet PMS is a pharmacy management ecosystem I designed and built for Minet Medicines Pharmacy, my community pharmacy in Anyinam, Eastern Region, Ghana. The system addresses a specific operational problem: managing inventory, tracking sales, and maintaining real-time oversight of a pharmacy in a rural, low-connectivity environment — without relying on expensive commercial software designed for larger urban operations.

The result is a three-part system — a web dashboard, a mobile companion app, and cloud infrastructure — working together to give me full operational visibility whether I'm on-site or thousands of miles away.

The Problem

Running a community pharmacy in rural eastern Ghana comes with challenges that off-the-shelf pharmacy software doesn't account for. Internet connectivity is intermittent. Power outages are routine. The staff operating the system have varying levels of technical comfort. And commercial pharmacy management tools are priced for urban chains, not single-location community pharmacies serving a local population.

Before MiNet PMS, inventory tracking was manual, sales records were paper-based, and I had no way to monitor operations remotely — a critical gap when I was studying in Vancouver while the pharmacy continued operating in Anyinam.

The Solution

I designed MiNet PMS as a three-component ecosystem:

  • MiNet PMS Web Dashboard (React): The primary interface, running on a Mac at the pharmacy. Handles inventory management, sales recording, stock alerts, and daily reporting. Designed for the staff member operating it — with large touch targets, minimal navigation depth, and clear visual feedback for every action.
  • MiNet Live Mobile App (Flutter / Android): A companion app installed on a Samsung Tab A9 at the pharmacy and on my personal phone. Receives real-time push notifications for every new sale, provides a mobile view of inventory and daily summaries, and allows me to monitor operations from anywhere.
  • Firebase Cloud Backend: The infrastructure connecting everything: Firestore for real-time database sync, Firebase Authentication for secure access, Cloud Functions for server-side logic (stock threshold alerts, daily summaries), and Firebase Cloud Messaging (FCM) for push notifications. Running on Firebase's Blaze plan with project ID minet-live.

Design Process

This project started with a problem, not a technology choice. I mapped out the daily workflow at the pharmacy — what the staff does from opening to closing, where friction points exist, what information I need remotely, and what would break if connectivity dropped.

From that mapping, I defined the core requirements:

  • Sales must be recordable even during internet outages (offline-first)
  • I need to know about every sale in real time when connectivity exists
  • Stock levels must trigger alerts before items run out
  • The interface must be usable by someone with limited tech experience
  • The system must run on hardware we already own

I then designed the UX flows on paper — screen-by-screen, interaction-by-interaction — before directing the development process. The entire system was built using AI-assisted development tools, with me serving as the product designer and director: defining what to build, how it should work, and what the experience should feel like, then guiding the AI through implementation, testing, and iteration.

Key Design Decisions

  • Offline-first architecture: Sales and inventory changes are written to a local cache first, then synced to Firestore when connectivity returns. This means the pharmacy never stops functioning because of an internet outage.
  • Push notifications as a management layer: Rather than requiring me to check a dashboard, the system pushes every sale to my phone. This passive monitoring model means I'm always aware without actively watching.
  • Minimal navigation depth: The web dashboard uses a single-level navigation structure. The most common actions — recording a sale, checking stock, viewing today's summary — are all one tap from the home screen.
  • Familiar visual language: The interface uses clear labels, large text, and color-coded status indicators (green for in-stock, amber for low, red for out-of-stock) rather than abstract icons or technical terminology.

Outcome

MiNet PMS has been in daily production use at the pharmacy since deployment. Key outcomes:

  • Zero downtime during internet or power interruptions, thanks to offline-first design
  • Real-time remote monitoring — I receive push notifications for every transaction, giving me full visibility from Vancouver
  • Reduced stock-outs — automated threshold alerts flag low inventory before items run out
  • Staff adoption — the simplified interface required minimal training; the staff member operating it was comfortable within the first day

What This Project Demonstrates

This wasn't a theoretical exercise or a class project. It's a real system solving a real problem in a real business — one I own and operate. It demonstrates:

  • Product thinking: Identifying a genuine operational problem, defining requirements from user workflows, and designing a system that addresses the actual constraints (connectivity, literacy, hardware, cost)
  • Systems design: Architecting a multi-platform ecosystem where web, mobile, and cloud components work together seamlessly
  • UX for constrained environments: Designing for intermittent connectivity, non-technical users, and hardware limitations
  • AI-assisted development as a design tool: Using AI not as a black box, but as a collaborator — directing the build process while maintaining ownership of every product and design decision
  • Shipping and iterating: Getting a functional system into daily production use and refining it based on real-world feedback
